Sentiment Analysis in Network Graph using RNN (LSTM) – Theory Explanation

Aim

To perform Sentiment Analysis on text data present in a Network Graph using a Recurrent Neural Network (RNN) with LSTM.

The model predicts whether a message/review is:

Positive

Negative



---

1. Introduction to Sentiment Analysis

Sentiment Analysis is a Natural Language Processing (NLP) technique used to determine emotions or opinions from text.

It identifies whether text expresses:

Positive sentiment

Negative sentiment

Neutral sentiment


Examples:

“Amazing product” → Positive

“Worst experience” → Negative



---

2. What is a Network Graph?

A Graph Theory graph is a structure consisting of:

Nodes (vertices)

Edges (connections)


In this program:

Each node contains a text review/message

Edges represent relationships between nodes



---

3. Graph Creation using NetworkX

G = nx.Graph()

Creates an undirected graph.


---

4. Adding Nodes

G.add_node(1, text="I absolutely love this product")

Each node stores:

Node ID

Text review


Example:

Node	Review

1	Positive review
2	Negative review


The graph contains 15 text-based nodes.


---

5. Adding Edges

G.add_edges_from([...])

Edges connect related nodes.

Purpose:

Represent relationships between reviews/messages

Form a connected network graph


Example:

(1,2)

means:

Node 1 is connected to Node 2



---

6. Text Extraction

texts = [G.nodes[1]['text'], ...]

All text reviews are extracted from graph nodes.

These texts become the input dataset for sentiment analysis.


---

7. Labels

labels = [1,0,0,1,...]

Labels represent sentiment classes.


---

Label Meaning

Label	Sentiment

1	Positive
0	Negative


Example:

“Excellent quality” → 1

“Poor design” → 0


This is a Binary Classification Problem.


---

8. Tokenization

tokenizer = Tokenizer(num_words=500)

Text cannot be processed directly by neural networks.

Tokenization converts words into numbers.


---

Example

Sentence:

"I love this product"

may become:

[1, 5, 7, 12]

Each word gets a unique integer index.


---

9. Converting Text to Sequences

texts_to_sequences(texts)

Transforms all reviews into numerical sequences.

These sequences are understandable by deep learning models.


---

10. Padding Sequences

pad_sequences(sequences, maxlen=10)

Reviews have different lengths.

Padding makes all sequences equal length.


---

Why Padding is Needed

Neural networks require fixed-size input.

Padding:

adds zeros to shorter sequences

truncates longer sequences


Final length:

maxlen = 10


---

11. RNN and LSTM

The model uses an Long Short-Term Memory layer.


---

12. Why LSTM is Used

Traditional RNNs struggle to remember long-term information.

LSTM solves this using memory cells and gates.

Benefits:

Remembers important words

Understands sentence context

Better for NLP tasks



---

13. Model Architecture

The model contains:

1. Embedding Layer


2. LSTM Layer


3. Dense Output Layer




---

14. Embedding Layer

Embedding(input_dim=5000, output_dim=64)

Purpose:

Convert words into dense vectors


Instead of simple numbers, words become meaningful vector representations.


---

Input Dimension

input_dim = 5000

Maximum vocabulary size.


---

Output Dimension

output_dim = 64

Each word is represented using a 64-dimensional vector.


---

15. LSTM Layer

LSTM(64)

This is the core sequence learning layer.


---

Purpose of LSTM

LSTM processes text word-by-word while remembering context.

Example:

Sentence:

> “The product is not good”



LSTM remembers:

not

which changes sentiment meaning.


---

64 Units

The LSTM contains 64 memory cells.

More units:

improve learning ability

increase computational complexity



---

16. Output Layer

Dense(1, activation='sigmoid')

Used for binary classification.

Only one neuron is needed because output is:

Positive

Negative



---

17. Sigmoid Activation Function

\sigma(x)=\frac{1}{1+e^{-x}}

Converts output into probability between:

0 and 1

Interpretation:

Closer to 1 → Positive

Closer to 0 → Negative



---

18. Model Compilation

model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['accuracy']
)


---

a) Adam Optimizer

Adam is an adaptive optimization algorithm.

Benefits:

Fast learning

Efficient weight updates

Better convergence



---

b) Binary Crossentropy Loss

Used for binary classification problems.

Measures prediction error.

Lower loss means better predictions.


---

19. Model Training

model.fit(X, y, epochs=5)

Training process:

1. Text sequences are fed into model


2. Embeddings are generated


3. LSTM learns patterns


4. Predictions are made


5. Errors are calculated


6. Weights are updated




---

Epochs

epochs = 5

One epoch means:

> One complete pass through the training dataset.
