⚖️ What is Load Balancing?

Imagine a restaurant with 3 waiters. If all customers go to only one waiter, that waiter gets overwhelmed while others stand idle. A manager (load balancer) distributes customers evenly among all waiters.

Without Load Balancer:        With Load Balancer:

Requests →  Server1 😵         Request1 → Server1 😊
            Server2 😴         Request2 → Server2 😊
            Server3 😴         Request3 → Server3 😊
                               Request4 → Server1 😊

🔄 Algorithm 1 — Round Robin Load Balancer

What is Round Robin?

Like dealing cards — give one to each player, in order, repeatedly:

Request 1 → Server1
Request 2 → Server2
Request 3 → Server3
Request 4 → Server1  ← wraps back to start
Request 5 → Server2
Request 6 → Server3
...

Every server gets an equal, fair, predictable turn.

class RoundRobinLoadBalancer:
  • class → creates a blueprint for the Round Robin load balancer
  • Groups related data and functions together
    def __init__(self, servers):
        self.servers = servers
        self.current_index = 0
  • __init__ → constructor — runs automatically when object is created
  • self.servers = servers → saves the server list inside the object
  • self.current_index = 0 → a pointer that tracks whose turn it is
    • Starts at 0 → first turn goes to servers[0] = "Server1"
Initial state:
  servers       = ["Server1", "Server2", "Server3"]
  current_index =      0
                        ↑
                   points here
    def get_server(self):
        server = self.servers[self.current_index]
        self.current_index = (self.current_index + 1) % len(self.servers)
        return server

This is the core logic — let's break it word by word:

        server = self.servers[self.current_index]
  • Picks the server at the current index position
  • current_index = 0 → picks servers[0] = "Server1"
        self.current_index = (self.current_index + 1) % len(self.servers)

This is the wrap-around trick using modulo %:

  • len(self.servers) = 3 (total servers)
  • % = remainder after division — forces index to stay within 0,1,2
Step-by-step:
  index=0 → (0+1) % 3 = 1 % 3 = 1  ✅
  index=1 → (1+1) % 3 = 2 % 3 = 2  ✅
  index=2 → (2+1) % 3 = 3 % 3 = 0  ✅ wraps back to 0!
  index=0 → cycle repeats...

Without %, index would go 0,1,2,3,4,5... and crash with IndexError!

        return server

Sends the chosen server name back to whoever called get_server().

🎲 Algorithm 2 — Random Load Balancer

What is Random Load Balancing?

Like rolling a dice — each request picks a server completely randomly. No tracking, no order — pure chance.

Request 1 → Server3  (random)
Request 2 → Server1  (random)
Request 3 → Server1  (random)  ← same server twice, that's okay
Request 4 → Server2  (random)
...
class RandomLoadBalancer:
    def __init__(self, servers):
        self.servers = servers
  • Much simpler than Round Robin — no index needed!
  • Just saves the server list
    def get_server(self):
        return random.choice(self.servers)
  • random.choice(list) → picks one random element from a list
  • Every call is completely independent — no memory of past picks
  • Each server has equal 1/3 = 33.3% chance every time
random.choice(["Server1", "Server2", "Server3"])
       │
       ▼
  Rolls random number internally
       │
       ▼
  Returns one of: "Server1", "Server2", or "Server3"

🚦 simulate_requests() Function

def simulate_requests(load_balancer, num_requests):
  • load_balancer → accepts any load balancer object (Round Robin or Random)
  • num_requests → how many requests to simulate
  • This is called polymorphism — one function works with multiple types!
    for i in range(1, num_requests + 1):
  • range(1, num_requests + 1) → generates numbers 1 to 10 (not 0 to 9)
  • +1 because range() stops before the end value
  • i = request number (1, 2, 3 ... 10)
        server = load_balancer.get_server()
        print(f"Request {i} routed to {server}")
  • load_balancer.get_server() → calls whichever algorithm's method
    • If Round Robin object → uses Round Robin logic
    • If Random object → uses Random logic
  • f"..." → f-string (formatted string) — inserts variable values directly
    • {i} → replaced by request number (1, 2, 3...)
    • {server} → replaced by server name ("Server1", etc.)
Output example:
  Request 1 routed to Server1
  Request 2 routed to Server2
