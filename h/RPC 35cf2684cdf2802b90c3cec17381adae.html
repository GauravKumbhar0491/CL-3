RPC

Let me break down this RPC (Remote Procedure Call) application in very simple language.

🌐 What is RPC?

RPC = Remote Procedure Call

Imagine you have a calculator in another room. You shout a number through the wall, and someone on the other side calculates it and shouts the answer back. That's RPC!

YOUR COMPUTER (Client)          ANOTHER COMPUTER (Server)
      │                                   │
      │  ── "Calculate factorial(5)!" ──► │
      │                                   │  computes 120
      │  ◄── "Answer is 120!" ──────────  │
      │                                   │

This code has two separate programs:

  • 🖥️ Server → does the math (factorial)
  • 💻 Client → sends a number and gets the answer

🖥️ SERVER SIDE — Full Breakdown

from xmlrpc.server import SimpleXMLRPCServer
  • xmlrpc → a built-in Python library for RPC communication
  • XML → the data format used to send messages between client and server (like a language both sides understand)
  • SimpleXMLRPCServer → a ready-made server class that handles all the network communication for you
server = SimpleXMLRPCServer(("localhost", 1234))

Creates a server and tells it where to listen:

PartMeaning
"localhost"Run on this same computer (IP: 127.0.0.1)
1234Listen on port 1234 (like a door number in a building)

Think of it like opening a shop at "Building: localhost, Door: 1234"

@server.register_function
def factorial(n):
  • def factorial(n) → defines a normal Python function
  • @server.register_function → this decorator tells the server: "Hey, make this function available for clients to call remotely"
  • Without this decorator, the function would only work locally — no client could access it
    if n < 0:
        return "Error: Factorial of negative number not possible"
  • Factorial is only defined for non-negative numbers
  • 0! = 1, 1! = 1, 5! = 120 ... but (-3)! = ❌ undefined
  • Returns an error message string if someone sends a negative number
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

Calculates factorial using a loop:

factorial(5):
  result = 1
  i=1 → result = 1 × 1 = 1
  i=2 → result = 1 × 2 = 2
  i=3 → result = 2 × 3 = 6
  i=4 → result = 6 × 4 = 24
  i=5 → result = 24 × 5 = 120
  return 120
  • range(1, n+1) → generates numbers 1, 2, 3 ... n
  • n+1 because range() stops before the end value
  • result *= i → shorthand for result = result * i
print("RPC Server running on port 1234...")
server.serve_forever()
  • print(...) → just a message so you know the server started
  • serve_forever() → keeps the server running non-stop, waiting for clients
    • Like a restaurant that never closes
    • Stops only when you press Ctrl+C

💻 CLIENT SIDE — Full Breakdown

import xmlrpc.client
  • The client version of the xmlrpc library
  • Server uses xmlrpc.server, Client uses xmlrpc.client — different roles, same protocol
server = xmlrpc.client.ServerProxy("http://localhost:1234/")
  • ServerProxy → creates a fake/proxy object that represents the remote server
  • "http://localhost:1234/" → address of the server
    • http:// → communication protocol
    • localhost → server is on the same machine
    • 1234 → port number (must match the server!)
🔑 Key idea: After this line, server looks like a normal Python object, but every method you call on it actually travels over the network to the real server!
num = int(input("Enter a number: "))
  • input(...) → waits for the user to type something (returns a string)
  • int(...) → converts that string to an integer
    • e.g., user types "5" → becomes 5
result = server.factorial(num)

This one line does a lot behind the scenes:

1. Client packages num=5 into XML format
2. Sends HTTP request to http://localhost:1234/
3. Server receives it, runs factorial(5)
4. Server packages result=120 into XML
5. Sends response back to client
6. Client unpacks XML → result = 120

To you, it looks like a simple function call. But secretly it's a full network round-trip!

print("Factorial received from server:", result)

Prints the final answer received from the server.

📦 What is XML in XMLRPC?

When client calls server.factorial(5), the data is sent as XML (a text format):

<!-- Request sent by client -->
<?xml version="1.0"?>
<methodCall>
  <methodName>factorial</methodName>
  <params>
    <param><value><int>5</int></value></param>
  </params>
</methodCall>

<!-- Response sent by server -->
<?xml version="1.0"?>
<methodResponse>
  <params>
    <param><value><int>120</int></value></param>
  </params>
</methodResponse>

Python handles all this XML automatically — you never have to write it yourself!
