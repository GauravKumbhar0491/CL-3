DEAP Evolutionary Algorithm — Line by Line Explanation

I'll explain this code in very simple language, just like before. Think of it like this: Imagine you are lost in a hilly area and want to find the lowest point (the deepest valley). You can't see the whole map. So you send out 50 explorers in random directions. After a while, you keep the best explorers (those who found lower ground), let them "have children" who explore nearby areas, occasionally let one wander off randomly (mutation), and repeat. Eventually, your explorers all gather at the lowest valley. 🏞️

This is exactly what DEAP (Distributed Evolutionary Algorithms in Python) helps us do — but with numbers instead of explorers!

🌱 Big Picture First

  • Goal of this code: Minimize the Sphere function → f(x) = x₁² + x₂² + x₃² + ... + x₁₀²
  • Correct answer: All x values should be 0, giving the minimum fitness = 0.
  • DEAP is a Python library that gives us ready-made tools to build Evolutionary Algorithms (like Genetic Algorithms).

🎯 Section 2: The Objective (Evaluate) Function

  • individual = a list of 10 numbers, e.g., [1.2, -3.5, 0.7, 2.1, ..., -1.4].
  • For each number x, calculate x², then add them all up.
  • Example: if individual = [1, 2, 3], then fitness = 1² + 2² + 3² = 14.
  • Lower value = better (we want the sum of squares to be as small as possible).
  • ⚠️ Notice the comma at the end → return sum(...), makes it a tuple. DEAP requires fitness to be a tuple (even if it has just one value), because some problems have multiple objectives.

🏗️ Section 3: Creating Custom Classes

  • Creates a new class called FitnessMin.
  • weights=(-1.0,) → the -1.0 tells DEAP we want to minimize fitness.
  • +1.0 would mean maximize.
  • The comma , again makes it a tuple.
  • Creates an Individual class that behaves like a Python list but also has a fitness attached to it.
  • So an individual is a list of 10 numbers + a fitness score.

🧰 Section 4: Initializing the Toolbox

  • Toolbox = a container where we register all our GA functions (like a labeled drawer for each tool).

🧪 Section 5: Defining Genes (Attributes)

  • Registers a function called attr_float that produces a random decimal number between -5 and 5.
  • This is how each gene in an individual is created.

👤 Section 6: Building Individuals & Population

  • initRepeat → repeat the attr_float function 10 times to create one individual.
  • So each individual is a list of 10 random floats, e.g., [1.2, -3.5, 0.7, 2.1, -4.6, 0.3, 3.9, -2.0, 1.1, -0.5].
  • This is a 10-dimensional problem (10 variables to optimize).
  • Creates a population — a list of many individuals.
  • We'll specify the population size later when we call this.

🔧 Section 7: Registering Operators

  • mate = Crossover (mixing two parents to make children).
  • cxBlend = Blend Crossover → instead of just swapping genes, it creates children with values blended between the parents.
  • Formula: child = parent1 + α × (parent2 - parent1), with random variation.
  • alpha=0.5 → controls how much "blending" happens. Higher alpha = more exploration.
  • mutate = adds random changes to genes.
  • mutGaussian → mutations come from a Gaussian (bell curve) distribution.
  • mu=0 → mean of the random change is 0 (changes are usually small).
  • sigma=1 → standard deviation = 1 (size of typical changes).
  • indpb=0.2 → each gene has a 20% chance of being mutated.
  • select = how to pick survivors for the next generation.
  • selTournament → pick 3 random individuals, keep the best one. Repeat to fill the new population.
  • This gives weaker individuals a small chance to survive too (helps diversity).
  • Registers our evaluate function (the Sphere function) as the fitness measure.

🚀 Section 8: The Main Function

  • Creates a population of 50 random individuals (each is a list of 10 floats between -5 and 5).

📊 Section 9: Statistics Setup

  • Creates a Statistics object that will collect data each generation.
  • lambda ind: ind.fitness.values → tells it to look at each individual's fitness value.
  • Registers 4 statistics to track every generation:
StatMeaning
avgAverage fitness of population
stdStandard deviation (how spread out the fitness values are)
minBest (lowest) fitness in this generation
maxWorst (highest) fitness in this generation

These help us watch the algorithm learn over time.

🔄 Section 10: Running the Algorithm

  • eaSimple = Evolutionary Algorithm Simple — the standard evolution loop. It does:
  • Evaluate everyone.
  • Select parents.
  • Apply crossover and mutation.
  • Replace the population.
  • Repeat for many generations.
ParameterMeaning

It returns:

- result → the final evolved population.

- logbook → a record of all generations' statistics.

🏆 Section 11: Print the Best Individual

  • selBest(result, k=1) → picks the top 1 individual from the final population.
  • [0] → gets the first (and only) item from that list.
  • Prints the best 10-number solution and its fitness.
  • Expected output: all 10 numbers should be very close to 0, and fitness close to 0.
  • Returns the final population and the logbook so we can analyze them later (e.g., plot graphs).

▶️ Section 12: Running the Code

  • Calls main() to run everything.

✨ Why DEAP is Powerful

  • Flexible: You can plug in any fitness function, mutation, or crossover.
  • Modular: Each piece (Individual, Toolbox, Operators) is independent and reusable.
  • Scalable: Supports distributed/parallel processing (that's where the "D" in DEAP comes from!).
  • Well-tested: Comes with many ready-made algorithms (eaSimple, eaMuPlusLambda, eaMuCommaLambda, etc.).
