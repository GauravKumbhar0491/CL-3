Hybrid Genetic Algorithm + Neural Network — Line by Line Explanation

I'll explain this code in very simple language. Think of it like this: We are trying to build a "brain" (Neural Network) that can identify flower types. But the brain has some settings (knobs) that we need to tune. Instead of guessing those settings, we use a Genetic Algorithm (GA) — which works like natural evolution — to find the best settings automatically.

🌱 Big Picture First

  • Neural Network (NN) = A computer model that learns patterns (like a mini brain).
  • Genetic Algorithm (GA) = A search method inspired by Darwin's evolution — "survival of the fittest." It creates many random solutions, picks the best, mixes them, mutates them, and repeats until the best solution emerges.
  • Hybrid GA-NN = Using GA to find the best settings for the NN.

🌸 Section 2: Loading the Iris Dataset

  • Iris dataset = A famous dataset with 150 flowers belonging to 3 species (Setosa, Versicolor, Virginica).
  • X = Features (inputs) → sepal length, sepal width, petal length, petal width (4 numbers per flower).
  • y = Target (output) → 0, 1, or 2 (the flower species).

⚖️ Section 3: Normalization

  • Normalize = squeezing all values into the range [0, 1].
  • Why? Because Neural Networks learn better when all input numbers are on the same scale (e.g., 5.1 cm and 0.2 cm should not have unequal influence just due to size).
  • fit_transform = first learns the min and max values, then transforms the data.

✂️ Section 4: Train-Test Split

  • Splits data: 80% for training, 20% for testing.
  • random_state=1 = a fixed seed so the split is the same every time (reproducible results).

🧬 Section 5: GA Setup with DEAP

  • FitnessMax = We tell DEAP we want to maximize fitness (higher score = better). The 1.0 weight means "maximize." If it were -1.0, it would minimize.
  • Individual = One candidate solution. It's a Python list (e.g., [10, 0.01, 0.001]) and it has a fitness attached to it.
  • Toolbox = A container where we register all the GA functions (like recipes in a cookbook).

🧪 Section 6: Defining Genes (Attributes)

Each individual has 3 genes (3 settings for the neural network):

  • random.randint → random whole number.

👤 Section 7: Creating an Individual & Population

  • initCycle = cycles through the three attribute functions once (n=1) to build one individual like [12, 0.05, 0.003].
  • population = a list of many individuals. initRepeat repeats individual creation many times.

🎯 Section 8: Fitness Function (The Most Important Part)

  • This function grades an individual.
  • round() rounds the hidden neurons to integer (since you can't have 7.3 neurons!).
  • max(...) and min(...) clamp values into safe ranges so mutation doesn't push them into invalid territory (e.g., negative learning rate).
  • Builds a Neural Network with these settings.
  • hidden_layer_sizes=(hidden,) → one hidden layer with that many neurons.
  • max_iter=1000 → train for up to 1000 iterations.
  • fit → train the NN on training data.
  • score → accuracy on test data (between 0 and 1).
  • Returns a tuple(score,) — DEAP requires fitness as a tuple even if it has just one value.

🛠️ Section 9: Registering GA Operators

  • mate = Crossover (mixing two parents to create children).
  • cxTwoPoint = "Two-point crossover" → picks 2 cut points in the gene list and swaps the middle section between two parents. Like mixing DNA.
  • mutate = Mutation (random small change in genes — like a DNA copy error).
  • mutGaussian → adds a random number from a Gaussian (bell curve) distribution.
  • mu=0 → mean of the noise is 0.
  • sigma=0.1 → standard deviation = 0.1 (size of the random nudge).
  • indpb=0.2 → each gene has a 20% chance of being mutated.
  • select = Selection of who survives to reproduce.
  • selTournament → pick 3 random individuals, the best one wins. Repeat to fill the next generation. This is called "tournament selection."

🚀 Section 10: Running the GA

  • Creates an initial population of 20 random individuals.
  • eaSimple = Evolutionary Algorithm Simple → the standard GA loop:
  • Evaluate everyone.
  • Select parents.
  • Crossover & mutate to make children.
  • Replace old population.
  • Repeat for ngen generations.
  • cxpb=0.7 → 70% chance two selected parents will crossover.
  • mutpb=0.2 → 20% chance an individual will be mutated.
  • ngen=50 → run for 50 generations.
  • verbose=False → don't print logs every generation.
  • After 50 generations, pick the best individual (highest accuracy).
  • Print the winning settings.

✨ Why Is It Called "Hybrid"?

Because two techniques work together:

1. Neural Network does the actual classification (learning to recognize flowers).

2. Genetic Algorithm searches for the best hyperparameters (knobs) for that neural network.

Alone, you would manually guess settings like hidden neurons = 10, lr = 0.01. The GA does this guessing automatically and intelligently through evolution — saving time and giving better accuracy.
